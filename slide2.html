Part 1: Problem Definition (10%)

Problem Statement & Context

The Business Bottleneck

Customer feedback is highly valuable but severely under-utilised due to the massive volume of manual review required. Businesses struggle to extract actionable insights quickly from text data.

Our Solution

An automated, production-grade REST service that classifies sentiment and extracts Aspect-Based Sentiment Analysis (ABSA) in real-time, evaluated on the SemEval-2014 Restaurant Reviews dataset.

Slide 2 / 21
Part 1: User Requirements (10%)

Core Use Cases & Scope

In Scope

  • Sentence-level: Positive, Negative, Neutral classification.
  • ABSA: Fine-grained aspects (Food, Service, Ambiance, Price).
  • Operations: Real-time API & Batch CSV processing.
  • Explainability: Token-level SHAP attributions.

Out of Scope / Constraints

  • No user authentication or rate limiting implemented.
  • English language only (Language guard active).
  • Constraint: CPU-only Docker image. Offline model serving required.

Part 2: Technical Deep Dive (DevOps)

Multi-stage Docker Build

  • Stage 1 (Builder): python:3.11-slim. Installs heavy PyTorch CPU wheels and requirements.
  • Stage 2 (Runtime): Copies only site-packages to drastically minimize the final image size.
  • Offline Enforcement: Hugging Face model weights are downloaded at build time. TRANSFORMERS_OFFLINE=1 ensures no network calls.
# Stage 1: build FROM python:3.11-slim AS builder RUN pip install torch --index-url ... # Stage 2: lean runtime FROM python:3.11-slim COPY --from=builder /usr/local/lib/python... COPY src/ contracts/ RUN python src/model/download_models.py ENV TRANSFORMERS_OFFLINE=1 CMD ["uvicorn", "src.main:app"]

Part 2: Technical Deep Dive (DevOps)

CI/CD Pipeline (GitHub Actions)

Automated workflow ci.yml triggers on every push to main/develop and pull requests.

  • 1. Install: Dependencies (httpx, pytest-cov).
  • 2. Test & Coverage: Runs Pytest suite.
  • 3. Lint (Hard): Flake8 fails build instantly on syntax errors (E9) or undefined names (F82).
  • 4. Lint (Soft): Flake8 warns on complexity metrics.
jobs: test: runs-on: ubuntu-latest steps: - run: pip install -r requirements.txt - run: pytest --cov=src tests/ # Hard failure on syntax errors - run: flake8 --select=E9,F63,F7,F82 # Soft warning for complexity > 10 - run: flake8 --exit-zero --max-complexity=10

Part 2: Technical Deep Dive (Data)

Data Quality Validation

The DataQualityValidator

Ensures robust data entering the pipeline. Automatically generates quality_report.json logged to MLflow.

  • Schema Checks: Validates required columns (sentence_id, text, sentiment).
  • Referential Integrity: Ensures no orphan aspect rows exist.
  • Null Constraints: Fails pipeline if any column exceeds a 1% null rate.
  • Distribution Monitoring: Triggers warnings if any class falls below 5% representation.
